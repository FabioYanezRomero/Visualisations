Análisis Técnico de flujo DPS RESUME (Punto por Punto)

1. Estado Control Plane (RESUMED/STARTED): El Control Plane transiciona a RESUMED al recibir la orden. Sin embargo, para efectos del Data Plane, esto es un retorno al estado operativo STARTED. El sistema busca el Data Plane que originalmente manejaba este proceso.

2. Reutilización de Mensaje (Idempotencia): El protocolo DPS es idempotente. Enviar un DataFlowStartMessage para un proceso que ya existe (y estaba suspendido) le indica al Data Plane que debe actualizar el estado a activo. No se duplican recursos, se reactivan.

3. Regeneración de Credenciales (Caso PULL): Este es el punto crítico. Si el flujo es Consumer PULL, la respuesta del paso 8 incluirá una nueva DataAddress con un nuevo token de acceso. El consumidor debe descartar la referencia anterior (EDR) y usar esta nueva para volver a descargar datos.
