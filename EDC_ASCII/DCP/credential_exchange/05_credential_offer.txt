5. Credential Offer (Oferta de Credencial)

   ISSUER (SERVIDOR)                                HOLDER (CLIENTE)
   +-----------------------+                        +-----------------------+
   | Issuer Service        |                        | Identity Hub          |
   |                       |  (1) Offer Message     |                       |
   | [IssuerCredential     | ---------------------> | [Credential Offer API]|
   |  OfferService]        |   POST /offers         |                       |
   | (Crea oferta firmada) |                        |  (2) Validation       |
   |                       |                        | [CredentialOffer      |
   |                       |  (3) Acknowledge       |  Handler]             |
   |                       | <--------------------- | (Verifica firma Issuer|
   |                       |   HTTP 200 OK          |  usando DID)          |
   |                       |                        | (Guarda en OfferStore)|
   +-----------------------+                        +-----------------------+

Descripción Técnica:

El Issuer notifica proactivamente al Holder que una credencial está disponible (ej. una renovación o un beneficio de lealtad).

(1) Offer Message: El IssuerCredentialOfferService genera un mensaje CredentialOfferMessage. Este incluye el tipo de credencial ofrecida (ej. MembershipCredential), el esquema y el DID del emisor.

(2) Validation & Store: El Identity Hub del Holder recibe el mensaje. Utiliza el DcpIssuerTokenVerifier para validar criptográficamente que la oferta proviene realmente del Issuer declarado. Si es válida, se persiste en el CredentialOfferStore.

(3) Acknowledge: Se confirma la recepción. Esto suele disparar un evento interno (CredentialOfferObservable) para que el usuario o un proceso automático decida si inicia el flujo de solicitud (Diagrama 2)
